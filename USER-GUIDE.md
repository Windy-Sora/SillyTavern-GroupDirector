## 1. 快速开始（5 分钟上手）

### 安装

将 `SillyTavern-GroupDirector` 文件夹放入 ST 的 `public/scripts/extensions/third-party/` 目录，重启 ST。在扩展管理面板中启用。

### 三步启动

**第一步：选模式** → 打开抽屉 1「模式与开始」，选择 LLM 模式（效果更好）或公式模式（零 token）。

**第二步：准备角色** → 快速启动区，点击「全部重新生成」生成角色档案，点击「提取全部」生成角色记忆。

**第三步：开始对话** → 进入群聊，正常发送消息。角色不再抢话，按导演顺序发言。

### 更快的方式

配置档管理中从预设加载 `group-director-default` → 一键应用——所有推荐配置即刻生效。

## 2. 抽屉指南（逐控件详解）

### 2.1 模式与开始

整个插件的总入口。分为四块：模式选择、快速启动、公式配置、配置档管理。

#### 模式选择

| 控件 | 类型 | 作用 |
|------|------|------|
| 语言 / Language | 下拉框 | 切换中英文界面。即时生效 |
| **关闭** | 单选按钮 | 不干预 ST 默认行为。选此项后插件完全不参与群聊生成 |
| **公式判断** | 单选按钮 | 本地评分模式。用关键词、提及、主动性等计算每个角色分数，Top-N 允许发言。不调用任何 LLM |
| **大模型判断** | 单选按钮 | LLM 导演模式。每轮额外调用一次模型，由 AI 分析上下文决定谁发言、说什么。效果最好但消耗 token |
| 调试日志 | 勾选框 | 开启后在浏览器 Console 输出详细运行日志。排查问题时开启 |

#### 快速启动区

进入对话前一站式准备所有角色数据的区域。**与对应抽屉中的控件完全同步**。

**系统就绪灯**：顶部一行小字，显示档案、记忆、世界书三项的状态。🟢 表示就绪，🟡 表示待处理。

**角色档案板块**：

| 控件 | 动作 | 效果 |
|------|------|------|
| 启用角色档案 | 勾选 | 开启/关闭角色档案系统。关闭后不注入档案到 Prompt |
| 全部重新生成 | 点击 | 为当前群聊所有角色调用 LLM 生成档案。生成中会显示状态（就绪/生成中/失败） |
| 扫描存档档案 | 点击 | 从聊天存档中重新读取已有档案，刷新下方列表。不会重新生成 |

**角色记忆板块**：

| 控件 | 动作 | 效果 |
|------|------|------|
| 启用角色记忆 | 勾选 | 开启/关闭角色记忆系统 |
| 提取全部 | 点击 | 为当前群聊所有角色调用 LLM，从对话中提取新记忆 |
| 扫描 | 点击 | 刷新记忆条数统计 |

**世界书板块**：

| 控件 | 动作 | 效果 |
|------|------|------|
| 勾选框列表 | 勾选/取消 | 选择哪些世界书条目会注入到导演 Prompt。勾选越多，导演知道的设定越多 |
| 刷新按钮 | 点击 | 重新加载世界书列表（新增或改名后使用） |

**一键操作**：

| 按钮 | 效果 |
|------|------|
| 一键总结 | 调用 LLM 将当前聊天上下文压缩为一条摘要。降低后续对话的 token 消耗 |

#### 公式判断配置

仅在「公式判断」模式下生效。控制本地评分的具体参数。

**Top-N 设置**：

| 控件 | 说明 |
|------|------|
| 每轮允许发言人数 (Top-N) | 得分最高的前 N 个角色允许说话。设为 1 就是每次只有最相关的人发言 |
| 分析最近消息条数 | 评分时看最近几条消息来计算提及和近期发言。越大越准确但越慢 |
| 连续发言惩罚分（每条） | 上轮刚说过话的角色被扣分。越大越不容易连续抢话 |

**评分权重**：

| 权重 | 作用 |
|------|------|
| 名字被提及权重 | 在最近消息中被提到名字的角色获得加分。越高越倾向"被点名就说话" |
| 关键词匹配权重 | 角色卡描述中的关键词出现在对话中时加分。越高越容易被"触发" |
| 近期未发言加权 | 很久没说话的角色获得额外加分。越高越倾向"轮流发言" |
| Talkativeness 权重 | 角色卡中的 talkativeness 值（0-1）影响发言积极性 |

**触发器引擎**：

| 控件 | 说明 |
|------|------|
| 启用关键词触发器 | 从角色描述中自动切出关键词，匹配到就给该角色加分 |
| 触发器命中加分 | 关键词命中一次的加分值 |

**主动性系统**：

| 控件 | 说明 |
|------|------|
| 启用主动性系统 | 每轮给每个角色随机加分（0~基础值），模拟"突然想说话" |
| 主动性基础值 | 随机加分的上限。设 0 = 角色完全被动等待点名 |

#### 配置档管理

**作用**：保存、导出、导入整套插件配置。可创建多套配置档按需切换。

**配置档列表**：每条配置档有三个操作按钮：

| 按钮 | 效果 |
|------|------|
| 应用 | 将所选配置档的设置覆盖当前设置。弹窗确认后生效，建议刷新页面 |
| 导出 | 打包配置档为 `.zip` 文件下载。含清单 JSON + 自定义 Provider/Capability 源码 |
| 删除 | 删除这条配置档记录 |

**预设下拉框**：加载插件内置的默认配置档（`group-director-default`），加载后出现在列表中，可再进一步自定义。

**保存当前为配置档**：

| 控件 | 说明 |
|------|------|
| 配置档名称 | 给这套配置起个名字，如「我的奇幻配置」 |
| 备注（可选） | 备忘说明，如「适合长篇叙事」 |
| 导出范围勾选框 | 7 个抽屉的勾选，选哪个抽屉导出哪个抽屉的设置。模式与评分不导出 |
| 保存为配置档 | 点击创建配置档 |
| 导入 .zip | 点击选择 `.zip` 配置档文件，导入到列表中 |

### 2.2 大模型行为

控制 LLM 导演的核心行为——Prompt、剧本、上下文注入。分为三个折叠子区。

#### 📊 LLM 参数与角色描述

| 控件 | 作用 |
|------|------|
| 每轮最多发言人数 | 限制每轮几个角色说话。设 2 表示最多 2 人，其余被阻止。推荐 2-3 |
| 传入上下文层数 | 给导演看最近 N 条消息。越大上下文越全，但 token 消耗越大。推荐 10 |
| 严格按 LLM 顺序发言 | 开启：导演决定发言顺序，插件接管 ST 激活循环，按导演决定的顺序逐人生成。关闭：导演只决定谁能说，ST 自己排顺序 |
| 角色描述控制 | 全量传入：角色卡描述全部给导演看。切片截断：只传前 N 个字符。推荐切片 200 |
| 切片长度 | 切片模式下的截断字符数 |

#### 📝 Prompt 与模板设置

| 控件 | 作用 |
|------|------|
| Director Prompt 模板 | **核心配置**。告诉导演"你是谁、你要做什么、返回什么格式"。使用占位符注入数据。模型必须返回 JSON |
| 恢复默认 Prompt | 点击重置 Prompt 模板为内置默认值 |
| 知识库 (Knowledge) | 存放不会被渲染的参考文本——其中的 `{{XXX}}` 原样发给 LLM。用于教 LLM 使用 DSL 语法。通过 `{{knowledge}}` 引用 |
| 启用递归渲染 | 占位符渲染出的文本中仍包含 `{{...}}` 时继续解析。推荐开启 |
| 最大递归轮数 | 限制递归次数防止无限循环。推荐 5 |
| 调试模式：保留未识别占位符 | 开启后拼写错误的 `{{charcters}}` 会原样保留方便排查。默认关闭 |

#### 🎬 剧本、账本与强制发言

| 控件 | 作用 |
|------|------|
| 启用导演剧本 | 开启后导演为每个发言角色输出一段场景剧本（如「Alice 应该注意到 Bob 的表情变化，语气变得柔和...」），注入到角色 prompt 中指导表演 |
| 剧本注入位置 | Prompt 开头：剧本在全文最前面，角色能看到但注意力弱。对话附近：剧本接近实际对话位置，干预更强 |
| 剧本要求提示 | 告诉导演你希望的剧本风格。留空只要求基本场景描述。例如「保持温馨治愈的氛围」 |
| 剧本注入包装模板 (Script Wrapper) | 控制剧本如何包裹后注入角色 prompt。`{{script}}` 替换为实际剧本内容。`{{charMemoryCurrent}}` 可注入当前角色记忆 |
| 恢复默认 | 点击重置包装模板为内置默认值 |
| **记录导演账本** | 每次 LLM 决策后把完整 JSON 保存到聊天元数据。跟随对话导出/分支。一般开启 |
| 清空当前导演账本 | 删除所有导演历史记录 |
| **使用导演历史** | 把记录的导演账本注入当前 prompt，让导演知道之前发生了什么。保持剧情连续性 |
| 连贯剧本模式 | 仅上一轮：只注入最近的导演决策。完整历史：注入 N 轮历史 |
| 历史轮数 | 完整历史模式下注入多少轮。设 0 = 全部 |
| 包装模板 | 控制历史 JSON 如何包裹。有「仅上一轮」和「完整历史」两套模板 |

**强制发言**：用户通过 ST 按钮或 `/trigger` 命令手动触发特定角色发言时的处理方式。

| 选项 | 效果 |
|------|------|
| 原生放行 | 弹窗确认后让 ST 自行处理。不注入剧本，不记账本 |
| 直接截停 | 完全阻止强制发言 |
| LLM 接管 | 调用模型为该角色生成剧本并计入账本。是最智能但最慢的选项 |
| 强制发言 Prompt 注入 | LLM 接管模式下，附加到导演 Prompt 的系统指令。`{charName}` 自动替换 |

### 2.3 世界书增强

**用途**：选择哪些 ST 世界书条目的内容注入到导演 Prompt 中，让导演了解世界观设定。

| 控件 | 作用 |
|------|------|
| 世界书勾选列表 | 每个世界书对应一个勾选框。勾选 = 导演知道该设定的内容。不勾选 = 导演完全不知道 |
| 全选 / 取消全选 | 快速操作所有条目 |
| 最大注入条目数 | 限制传给导演的条目总数。即使勾选了很多，也只传最重要的前 N 条。推荐 15-30 |
| 刷新世界书列表 | ST 的世界书变更后点击刷新 |

> 这个抽屉和 ST 自带的世界书编辑是互补关系——ST 负责编辑管理，GD 负责选哪些给导演看。

### 2.4 角色档案与数据

四个折叠子区，管理所有角色级别的数据资产。

#### ▼ 角色档案系统

提前分析每个角色的特质、动机、关系，生成结构化档案注入 Director Prompt。

| 控件 | 作用 |
|------|------|
| 启用角色档案 | 开启/关闭整个档案系统。关闭后 `{{character_profiles}}` 返回空 |
| 扫描当前存档 | 从聊天存档中读取已有档案，恢复之前生成的数据 |
| 检测角色变动 | 检测群聊中有角色加入或退出，弹窗让用户选择处理方式 |
| 档案 Token 预算 | 超过这个 token 数时，非活跃角色的档案会被压缩（只保留 summary 前 100 字） |
| 并发数 | 同时生成几个角色的档案。0=全部同时，1=逐个，N=每批 N 个 |
| **生成器 Prompt 模板** | 告诉 LLM 如何分析角色卡。占位符：`{{charName}}` `{{charDescription}}` `{{charPersonality}}` `{{charScenario}}` |
| 恢复默认 | 重置生成器 Prompt |
| **JSON Schema** | 定义 AI 返回的 JSON 格式。可自定义字段。默认：summary/tags/motivation/relationships |
| 恢复默认 Schema | 重置 Schema |
| **渲染模板** | 控制 `{{character_profiles}}` 的输出格式。每角色占位符：`{{name}}` `{{summary}}` `{{tags}}` `{{motivation}}` `{{relationships}}` |
| 恢复默认渲染模板 | 重置渲染模板 |
| 全部重新生成 | 一键为当前群聊所有角色生成档案 |
| 档案管理卡片 | 每个角色一张可展开卡片：显示状态、hash 是否过期、编辑/重生成/删除按钮 |

#### ▼ 角色记忆系统

从对话历史中提取角色的关键经历、情感变化，生成记忆条目。

| 控件 | 作用 |
|------|------|
| 启用角色记忆 | 开启/关闭记忆系统。关闭后 `{{charMemory}}` 返回空 |
| Token 预算 | 限制记忆文本的总长度 |
| **记忆提取 Prompt** | 告诉 LLM 如何从对话中提取记忆 |
| 恢复默认 | 重置提取 Prompt |
| **JSON Schema** | 定义记忆条目格式。默认：`{ event, mood }` |
| 恢复默认 Schema | 重置 Schema |
| **渲染模板** | 控制 `{{charMemory}}` 的输出格式 |
| 恢复默认 | 重置渲染模板 |
| **压缩 Prompt** | 压缩旧记忆时使用的 LLM Prompt。留空使用内置默认 |
| 恢复默认 | 重置压缩 Prompt |
| 压缩时保留最近条数 | 压缩时保留最近的 N 条原文，其余合并为 LLM 摘要 |
| 单角色记忆上限 | 超过此数量自动裁剪最旧条目。默认 200 |
| **记忆管理按钮**：刷新列表 / 扫描已有对话 / 提取选中 / 提取全部 / 压缩旧记忆 / 回退 / 重置全部 / 检测失联 |
| 记忆卡片 | 每个角色可展开：显示记忆条目列表、编辑/删除单条、压缩/回退操作 |

#### ▼ 身份锚定 (Identity Anchor)

自定义身份锚定提示词，统一管理档案/记忆/摘要的使用方式。

| 控件 | 作用 |
|------|------|
| 文本框 | 书写指令告诉 LLM 如何看待角色数据。内置默认模板含 DSL 用法示范 |
| 清空 | 恢复为内置默认模板 |

**使用方式**：在 Director Prompt 或 Script Wrapper 中放入 `{{identity}}`。不会自动注入。

#### ▼ NPC 生成系统

根据对话上下文批量生成 NPC。

| 控件 | 作用 |
|------|------|
| 启用 NPC 生成 | 开启/关闭 |
| 最大 NPC 数量 | 限制总数 |
| 每次生成数量 | 批量大小 |
| 生成开场白 | 是否让 AI 为 NPC 写第一句话 |
| 生成 Prompt | 自定义 Prompt。支持所有 Provider 占位符 |
| NPC 卡片操作 | 编辑信息 / 导入为 ST 角色卡 / 删除 |

### 2.5 上下文与账本

#### 上下文总结

将聊天上下文压缩为简洁摘要，降低 token 消耗。

| 控件 | 作用 |
|------|------|
| 扫描存档总结 | 从聊天存档中读取已有总结 |
| 清除已禁用 / 隐藏已禁用 | 管理总结列表 |
| 启用上下文总结 | 开启后 `{{chatSummary}}` 有内容 |
| 复用上次总结 | 勾选：在旧总结基础上追加新片段。取消：全量上下文重新总结 |
| 总结 Prompt | 自定义总结风格。留空使用内置默认 |
| 恢复默认 Prompt | 重置 |
| **执行总结** | 调用 LLM 生成当前对话的摘要 |
| **重新总结** | 重新生成最新一条总结 |
| **回退总结** | 撤销最新总结 |
| **重置全部** | 清空所有总结 |
| 总结结果 | 可直接编辑总结文本。点击保存编辑提交 |

#### 摘要导出 / 导入

| 控件 | 作用 |
|------|------|
| 导出当前摘要 | 将当前激活的总结导出为 JSON 文件 |
| 导入摘要文件 | 从 JSON 文件导入外部摘要。导入后出现在下方列表 |
| 勾选面板 | 控制哪些导入的摘要通过 `{{importedSummary}}` 注入 Prompt |

#### 导演账本浏览器

| 控件 | 作用 |
|------|------|
| 刷新 | 重新加载账本列表 |
| Raw 模式 | 切换 JSON 原文/易读格式 |
| 账本条目的编辑/删除 | 内联编辑，生成中锁死 |

### 2.6 多模态与更多拓展

**PostSpeech 系统**：角色发言后自动调用 LLM 确定应该触发的多模态能力（如 TTS 旁白、生成配图）。

| 控件 | 作用 |
|------|------|
| 每次发言后激活 PostSpeech | 每个角色说完话后调用一次 PostSpeech Agent |
| 每次发言 Prompt | 自定义 PostSpeech 分析 Prompt |
| 回合结束后激活 PostSpeech | 一轮群聊全部说完后调用一次 |
| 阻塞模式 | 开启：等 PostSpeech 处理完才继续对话。关闭：后台异步执行 |
| 已注册能力列表 | 启用/禁用各个多模态能力 |
| PostSpeech 决策记录 | 查看历史决策（随聊天持久化） |

> 开启后每次发言增加 1 次 LLM 调用，回合结束增加 1 次。适合沉浸式场景，日常使用建议关闭。

### 2.7 资产管理

#### 导入 Provider

| 控件 | 作用 |
|------|------|

## 3. 两种导演模式

### 公式判断（零 token）

```
角色得分 = 名字被提及 × mentionWeight（默认 30）
         + 关键词匹配 × triggerScore（默认 40）
         + 近期未发言 × recencyWeight（默认 20）
         − 连续发言 × consecutivePenalty（默认 15）
         + Talkativeness × talkativenessWeight（默认 10）
         + 随机主动性（每轮 0 ~ initiativeBaseScore）
```

得分最高的 Top-N 个角色允许发言。适合快速测试和轻量日常。

### 大模型判断

每轮额外调用一次 LLM。导演分析上下文后返回 JSON：

```json
{
  "speakers": ["Alice", "Bob"],
  "reason": "Alice was directly addressed, Bob has relevant knowledge",
  "scripts": { "Alice": "Notices Bob's hesitation...", "Bob": "Recalls the clue from earlier..." }
}
```

`speakers` 顺序 = 发言顺序。`scripts` 是可选字段（需开启导演剧本）。

## 5. DSL 路径查询语法

### 基本查询

```
{{?name:field.subfield}}          → 取嵌套字段
{{?name:array[0]}}                → 取数组第 1 项
{{?name:array[-1]}}               → 取数组最后 1 项
{{?name:[key=value].field}}       → 按 key 精确查找
```

### 实战示例

```
{{?character_profiles:Alice.summary}}
  → "一位身经百战的王国骑士，恪守荣誉准则..."

{{?character_profiles:$character.motivation}}
  → 在 Script Wrapper 中自动取当前角色的 motivation

{{?npcList:[name=老格里姆].description}}
  → 查找名为"老格里姆"的 NPC 的描述

{{?directorLedger:scripts.$character}}
  → 在 Script Wrapper 中取导演为当前角色分配的剧本

{{?worldBooks:allEntries[comment=地理与空间].content}}
  → 查找注释为"地理与空间"的世界书条目的内容
```

### 块循环

```
{{#provider:arrayPath}}
  对每个元素渲染此块。$it 为当前索引
{{/provider}}
```

示例：
```
{{#npcList:all}}
  - {{?npcList:all[$it].name}}: {{?npcList:all[$it].description}}
{{/npcList}}
```

## 7. 常见场景配方

### 奇幻 RPG 长篇叙事

```
模式：大模型判断 → 每轮发言 2-3 → 角色描述切片 200
启用导演剧本 ✓ → 剧本要求「史诗奇幻，注重角色内心」
启用连贯剧本 → 完整历史 10 轮
启用档案 ✓ → 启用记忆 ✓ → 世界书注入 ✓
发布推荐：把整套配置存为配置档
```

### 轻量日常聊天

```
模式：公式判断 → Top-N 2 → 名字提及 30 → 近期待遇 20 → 连续惩罚 15
关闭 LLM 导演、剧本、档案、记忆
```

### 悬疑推理

```
模式：大模型判断 → 每轮 2
导演 Prompt 追加「保持悬疑，每段发言都可能是线索」
剧本要求「保持神秘，不揭示真相」
启用记忆 ✓（追踪角色各自知道什么信息）
```

### 快速测试角色

```
公式判断 → Top-N 1
快速启动 → 全部重新生成 → 提取全部记忆 → 勾选世界书 → 开始
```

## 8. 常见问题

**Q: 导演不工作了？**
A: 检查模式不是「关闭」。LLM 模式检查 Director Agent 的 API 连接测试。

**Q: 角色档案为空？**
A: 需要先生成——抽屉 1 快速启动 → 启用档案 → 全部重新生成。

**Q: API 密钥会泄露吗？**
A: 导出配置档时 Agent API 密钥自动清空。导入后需重新填写。

**Q: 自定义 Prompt 名字被拒绝？**
A: 仅限 `a-z 0-9 _`。不能与内置 Provider（`recentMessages` 等）或 ST 宏（`user` `char` `time` 等 72 个）重名。

**Q: 怎么彻底还原配置？**
A: 抽屉 1 配置档管理 → 预设加载 `group-director-default` → 应用。

**Q: 记忆压缩和记忆导出的关系？**
A: 压缩用 LLM 将旧记忆合并为摘要。压缩记忆带标记，导出时可选跳过。
