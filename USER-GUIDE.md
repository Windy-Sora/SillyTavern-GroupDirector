## 1. 快速开始

### 安装

将 `SillyTavern-GroupDirector` 文件夹放入 ST 的 `public/scripts/extensions/third-party/` 目录。重启 ST，在扩展管理面板中启用 "Group Director"。

### 基础配置

1. 打开 **模式与开始** 抽屉
2. 选择模式：
   - **公式判断**：零成本，本地评分决定谁发言
   - **大模型判断**：调 LLM 做导演决策（效果更好，消耗 token）
3. 切到群聊，开始对话——角色不再抢话

### 推荐配置（一键）

打开 **模式与开始** → 在配置档管理中找到预设下拉框 → 选择 `group-director-default` → 加载 → 点击 **应用**。这是开箱即用的最佳配置。

## 2. 抽屉指南

### 2.1 模式与开始

**用途**：所有功能的入口和快速启动面板。

**包含**：

| 区块 | 功能 |
|------|------|
| 语言 / 模式选择 | 切换中英文、选择公式判断或大模型判断 |
| 快速启动 | 档案状态、记忆统计、世界书选择、一键总结 |
| 公式判断配置 | 评分权重、触发器引擎、主动性系统 |
| 配置档管理 | 保存/导入/导出整套插件配置 |

**快速启动区**提供进入对话前所需的核心操作：
- 档案：启用→生成→检查状态
- 记忆：启用→提取→查看条数
- 世界书：勾选要注入的设定
- 一键总结：压缩上下文降低 token 消耗

---

### 2.2 大模型行为

**用途**：控制 LLM 导演的行为——Prompt、剧本、上下文处理。

**三个子区**：

**📊 LLM 参数与角色描述**
| 设置项 | 说明 |
|------|------|
| 每轮最多发言人数 | 限制每轮几个人说话（推荐 2-3） |
| 传入上下文层数 | 给导演看最近多少条消息 |
| 严格按 LLM 顺序发言 | 开启后导演决定发言顺序，逐个生成 |
| 角色描述控制 | 全量传入还是截断（推荐截断 200 字符） |

**📝 Prompt 与模板设置**
| 设置项 | 说明 |
|------|------|
| Director Prompt 模板 | 导演的决策提示词，用占位符引用数据 |
| 知识库 | 教 LLM 使用 DSL 的参考文本 |
| 模板递归渲染 | 占位符嵌套解析（推荐开启，5 轮） |

**🎬 剧本、账本与强制发言**
| 设置项 | 说明 |
|------|------|
| 启用导演剧本 | 导演为每个发言角色生成场景指导 |
| 剧本注入位置 | Prompt 开头（弱干预）或对话附近（强干预） |
| 剧本要求提示 | 告诉导演你想要的剧本风格 |
| 导演账本 | 记录每轮导演决策，支持连贯模式 |
| 强制发言处理 | 原生放行 / 直接截停 / LLM 接管 |

**常用占位符**：`{{recentMessages}}` `{{characters}}` `{{character_profiles}}` `{{charMemory}}` `{{worldInfo}}` `{{identity}}` `{{knowledge}}`

### 2.3 世界书增强

**用途**：选择哪些世界书条目注入到导演 Prompt 中。

| 设置项 | 说明 |
|------|------|
| 世界书勾选列表 | 勾选要暴露给导演的世界书 |
| 最大注入条目数 | 限制传给导演的条目数量（推荐 15-30） |

### 2.4 角色档案与数据

**四个子区**：

**角色档案系统**：提前分析每个角色的特质、动机、关系，生成结构化档案注入 Prompt。
- 一键生成全部角色档案
- 自定义生成 Prompt / JSON Schema / 渲染模板
- 支持导出/导入（`.json` 含模板配置）
- 预设：`fantasy-rpg`

**角色记忆系统**：从对话中提取角色关键经历和情感变化。
- LLM 驱动的记忆提取
- 支持压缩（老记忆合并摘要，可自定义压缩 Prompt）
- 导出/导入（`.json`，含 4 项模板配置）
- 智能 avatar 匹配（同角色不同文件名自动识别）

**身份锚定**：自定义身份锚定提示词，统一管理档案/记忆/摘要的使用方式。
- 通过 `{{identity}}` 占位符注入
- 内置默认模板（含 DSL 用法示范）
- 完全 opt-in，不会自动注入

**NPC 生成系统**：根据上下文生成 NPC，支持一键导入为角色卡。
- 导出/导入（`.json` 含生成 Prompt）
- 预设：`npc-fantasy-tavern`

---

### 2.5 上下文与账本

**用途**：管理对话摘要和导演决策记录。

| 区块 | 功能 |
|------|------|
| 上下文总结 | 压缩聊天为摘要降低 token（支持复用/回退/编辑） |
| 摘要导出/导入 | 导出激活摘要或导入外部摘要（通过 `{{importedSummary}}` 注入） |
| 导演账本浏览器 | 查看、编辑、清空导演决策历史 |

---

### 2.6 多模态与更多拓展

**用途**：PostSpeech 多模态策略——角色发言后自动触发 TTS、图像等能力。

每轮增加 1-2 次 LLM 调用，适合需要沉浸式体验的场景。默认建议不开启。

---

### 2.7 资产管理

**用途**：管理自定义扩展模块。

| 区块 | 功能 |
|------|------|
| 导入 Provider | 导入自定义 `.js` Provider 文件，注册为新的 `{{placeholder}}` |
| 导入 Capability | 导入自定义 `.js` Capability 文件，扩展多模态能力 |
| 自定义 Prompt 模板 | 创建 `{{name}}` 占位符（命名仅限 a-z 0-9 _），可在任意模板中引用 |

**自定义 Prompt 说明**：
- 新建后立即注册为 `{{name}}` Provider
- 支持独立开关和总开关
- 同名导入自动检测冲突
- 命名时自动拦截 ST 内置宏冲突（72 个）

---

### 2.8 Agent 与工具

**用途**：为每个 Agent 单独配置 API（支持 OpenAI / Anthropic 协议）。

| 区块 | 功能 |
|------|------|
| Agent API 配置 | 每个 Agent 独立的端点、密钥、模型 |
| 执行追踪 | 查看 Agent 每次运行耗时和输出摘要 |
| 模板测试器 | 输入模板 + 占位符，查看实时渲染结果 |

**Agent 列表**：
- Director（导演决策）
- ForceSpeak（强制发言处理）
- Profile（档案生成）
- Summary（上下文总结）
- NPC（NPC 生成）
- Memory（记忆提取）
- PostSpeech（多模态策略）

## 3. 两种导演模式

### 公式判断（零 API 调用）

```
角色得分 = 名字被提及 × w_mention
         + 关键词匹配 × triggerScore
         + 近期未发言加权 × w_recency
         − 连续发言惩罚 × w_consecutive
         + Talkativeness × w_talkativeness
         + 随机主动性值
```

得分最高的 Top-N 个角色允许发言。适合快速测试和轻量场景。

### 大模型判断

每轮额外调用一次 LLM，由 LLM 分析上下文后输出 JSON：

```json
{
  "speakers": ["Alice", "Bob"],
  "reason": "Alice was directly addressed, Bob has relevant knowledge",
  "scripts": { "Alice": "...", "Bob": "..." }
}
```

`speakers` 顺序即是发言顺序。适合追求叙事质量的场景。

## 5. DSL 路径查询语法

在 `{{?name:path}}` 中，`path` 支持以下查询方式：

### 基本查询

```
{{?character_profiles:Alice.summary}}          → 取 Alice 的 summary
{{?directorLedger:speakers[0]}}                → 取第一个发言人
{{?npcList:[name=老格里姆].description}}       → 按 name 查找 NPC
{{?npcList:[-1].scenario}}                     → 最后一个 NPC 的场景
{{?directorLedger:events[-1].title}}           → 最新事件的标题
{{?worldBooks:allEntries[comment=地理].content}} → 按注释查找世界书
```

### 运行时变量

```
{{?character_profiles:$character.summary}}     → 当前发言角色的档案摘要
{{?character_profiles:$character.motivation}}  → 当前角色的动机
{{?character_profiles:$character.tags}}        → 当前角色的标签
```

### 块循环

```
{{#npcList:all}}
  {{?npcList:all[$it].name}}: {{?npcList:all[$it].description}}
{{/npcList}}
```

## 6. 导出与导入

Group Director 提供完整的资产可插拔能力。所有导出文件均为标准 JSON，可分享、可备份。

### 6.1 数据级导出（per-chat）

| 类型 | 位置 | 格式 | 粒度 |
|------|------|------|------|
| 角色档案 | 角色档案与数据 → 档案导出/导入 | `.json` | 逐角色勾选 |
| NPC | 角色档案与数据 → NPC 导出/导入 | `.json` | 逐条勾选 |
| 角色记忆 | 角色档案与数据 → 记忆导出/导入 | `.json` | 逐角色勾选 |
| 上下文摘要 | 上下文与账本 → 摘要导出/导入 | `.json` | 一键导出激活摘要 |

### 6.2 配置级导出（全局）

| 类型 | 位置 | 格式 | 说明 |
|------|------|------|------|
| 配置档 | 模式与开始 → 配置档管理 | `.zip` | 按抽屉选择范围 |
| 自定义 Prompt | 资产管理 → 自定义 Prompt 模板 | `.json` | 一键导出全部 |

### 6.3 配置档导出范围

保存配置档时按抽屉勾选：

| 抽屉 | 导出内容 |
|------|------|
| 大模型行为 | Prompt / Script / Wrapper / WorldInfo / ForceSpeak / Knowledge / Template |
| 世界书增强 | `worldBookMaxEntries`（不导出文件名列表）|
| 角色档案与数据 | Profile / Memory / Identity / NPC 全部设置 |
| 上下文与账本 | Summary / Continuity 设置 |
| 多模态 | PostSpeech 全部设置 |
| 资产管理 | User Providers/Capabilities 源码 + Custom Prompts |
| Agent 与工具 | AgentConfigs（**API 密钥自动清除**）+ Trace |

模式与评分不导出。

### 6.4 导入行为

- 同名数据默认保留现有、追加新数据
- 检测到冲突时弹窗确认
- 模板一致性通过 hash 比对自动检测，不一致时警告但允许继续

## 7. 常见场景配方

### 场景 1：奇幻 RPG 长篇叙事

```
模式: 大模型判断
每轮发言人数: 2-3
角色描述: 切片 200 字符
启用导演剧本: ✓
剧本要求: "保持史诗奇幻风格，注重角色内心戏"
启用连贯剧本: 完整历史，10 轮
启用角色档案: ✓
启用角色记忆: ✓
世界书注入: ✓
```

### 场景 2：轻量日常聊天

```
模式: 公式判断
Top-N: 2
名字提及权重: 30
近期未发言加权: 20
连续发言惩罚: 15
关键词触发器: ✓
```

### 场景 3：悬疑推理

```
模式: 大模型判断
每轮发言人数: 2
角色描述: 全量
导演 Prompt: 添加"保持悬疑氛围，每次发言都可能是线索"
启用导演剧本: ✓
剧本要求: "保持神秘感，不要过早揭示真相"
启用角色记忆: ✓（追踪谁能掌握什么信息）
```

### 场景 4：快速测试角色卡片

```
模式: 公式判断
Top-N: 1
停用导演剧本
快速启动 → 扫描存档档案 → 全部重新生成
快速启动 → 提取全部记忆
快速启动 → 勾选世界书
→ 开始对话
```

## 8. 常见问题

**Q: 导演不工作了？**
A: 检查抽屉 1 → 模式不是"关闭"。如果 LLM 模式没反应，检查 Director Agent 的 API 配置。

**Q: 角色档案为什么是空的？**
A: 需要先生成——抽屉 1 快速启动 → 启用档案 → 全部重新生成。每个角色会调用一次 LLM。

**Q: 导出模板后，导入到新群聊能用吗？**
A: 数据级导出（档案/记忆/NPC）依赖角色匹配，avatar 精确匹配 → 名称匹配 → 模糊匹配，一般能自动对应。配置级导出（配置档）不受群聊影响。

**Q: API 密钥会泄露吗？**
A: 导出配置档时 Agent API 密钥自动清除。导入后需要重新填写。

**Q: 为什么自定义 Prompt 名字被拒绝了？**
A: 名字只能用 `a-z 0-9 _`，且不能与内置 Provider 或 ST 宏重名（如 `user`、`char`、`time`）。

**Q: 记忆压缩和记忆导出的关系？**
A: 压缩会用 LLM 将旧记忆合并为一条摘要。压缩记忆带 `compressed: true` 标记。导出时可选跳过压缩记忆。

**Q: 怎么彻底还原？**
A: 模式与开始 → 配置档管理 → 从预设加载 `group-director-default` → 应用。这是开箱即用的推荐配置。
